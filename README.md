# jpa-shop

복잡한 비즈니스 요구사항을 반영하여 객체지향적인 도메인을 설계하고, 성능 최적화를 달성한 백엔드 시스템입니다.

## Tech Stack
- **Language:** Java
- **Framework:** Spring Boot, Spring Data JPA, QueryDSL
- **Database:** H2

## Key Features
- 상품 등록/수정/조회 및 재고 관리
- 회원 가입 및 주문 로직 구현
- 복잡한 도메인 간의 연관관계 매핑 및 영속성 전이(Cascade) 처리

## Technical Decisions & Troubleshooting

### 1. 객체 패러다임과 관계형 DB의 불일치 해결
- **다대다(N:M) 한계 극복:** 테이블 상에서 불가능한 다대다 관계를 `1:N`, `N:1` 관계로 풀어냈습니다.
- **상속 관계 매핑:** 객체의 상속 구조를 DB 테이블에 반영하기 위해 `SINGLE_TABLE` 전략을 사용하여 구현했습니다.
- **생명주기 관리:** 부모 엔티티가 자식 엔티티의 생명주기를 관리하도록 `Cascade.ALL`과 `orphanRemoval`을 적용했습니다.

### 2. JPA 동적 쿼리의 한계와 QueryDSL 도입
- **문제:** 검색 조건(이름, 상태 등)에 따라 달라지는 동적 쿼리를 `JPA Criteria`로 작성했으나, 코드가 복잡해져 유지보수성이 심각하게 저하되었습니다.
- **해결:** **QueryDSL** 통해 자바 코드로 쿼리를 작성했습니다. 이를 통해 코드 가독성을 높이고, 오타로 인한 에러를 컴파일 시점에 발견할 수 있게 되었습니다.

### 3. N+1 문제 발생과 Fetch Join 최적화
- **문제:** 엔티티 조회 시 연관된 엔티티를 가져오기 위해 수십 개의 추가 쿼리가 발생하는 N+1 문제가 발생했습니다.
- **해결:** 모든 연관관계를 **지연 로딩**(`FetchType.LAZY`)으로 변경하여 추가 쿼리를 방어했고 이후, 함께 출력되어야 하는 연관 데이터에 부분적으로 **페치 조인**을 적용하여 1번의 쿼리로 성능을 최적화했습니다
